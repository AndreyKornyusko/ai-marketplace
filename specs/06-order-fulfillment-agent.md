# Spec 06 — Order Fulfillment Agent

**Status:** DRAFT

**Implemented by:** db-agent (if schema), backend-agent (tools API), ai-agent-builder
**Reviewed by:** spec-checker, backend-reviewer, security-reviewer

---

## Overview

The OrderFulfillmentAgent is an AI agent that handles order-related customer intents:
checking order status, modifying quantities, cancelling orders, and initiating returns.

---

## Agent Responsibilities

- Answer customer questions about their order status
- Help customers modify orders (before PROCESSING status only)
- Help customers cancel orders (before SHIPPED status only)
- Initiate return/refund requests (after DELIVERED status)
- Escalate to human support when outside its authority

---

## Tools

### check_order_status

```typescript
{
  name: 'check_order_status',
  description: 'Get the current status and details of a customer order by order ID.',
  schema: z.object({
    orderId: z.string().uuid(),
    customerEmail: z.string().email().optional(),  // for guest order verification
  }),
}
```

### check_stock_batch

```typescript
{
  name: 'check_stock_batch',
  description: 'Check real-time stock levels for one or more products. MUST call before confirming any order change.',
  schema: z.object({
    productIds: z.array(z.string().uuid()),
  }),
}
```

### modify_order_quantity

```typescript
{
  name: 'modify_order_quantity',
  description: 'Modify the quantity of an item in a PENDING or CONFIRMED order. Requires stock validation first.',
  schema: z.object({
    orderId: z.string().uuid(),
    orderItemId: z.string().uuid(),
    newQuantity: z.number().int().min(1),
  }),
}
```

### cancel_order

```typescript
{
  name: 'cancel_order',
  description: 'Cancel an order. Only possible if status is PENDING or CONFIRMED.',
  schema: z.object({
    orderId: z.string().uuid(),
    reason: z.string(),
  }),
}
```

### initiate_return

```typescript
{
  name: 'initiate_return',
  description: 'Initiate a return request for a delivered order.',
  schema: z.object({
    orderId: z.string().uuid(),
    itemIds: z.array(z.string().uuid()),
    reason: z.string(),
  }),
}
```

### escalate_to_human

```typescript
{
  name: 'escalate_to_human',
  description: 'Escalate the conversation to a human support agent when the issue is outside AI authority.',
  schema: z.object({
    reason: z.string(),
    summary: z.string(),  // brief summary of the conversation
  }),
}
```

## Grounding Requirements

- MUST call `check_stock_batch` before confirming any `modify_order_quantity`
- MUST validate order belongs to the authenticated customer before any mutation
- MUST check order status allows the requested action before calling mutation tools

---

## System Prompt Requirements

Include in system prompt:
1. Store policies (from `specs/01-store-context.md`)
2. What the agent can and cannot do
3. Escalation triggers (see below)
4. Grounding rules (from `skills/grounding-patterns.md`)

### Escalation Triggers

Agent MUST call `escalate_to_human` when:
- Customer claims they were charged incorrectly
- Order disputes or chargeback requests
- Customer is abusive or threatening
- Agent cannot resolve the issue in 3 attempts
- Issue involves fraud or security concerns

## Acceptance Criteria

1. Agent can look up order status without hallucinating data
2. Agent refuses to modify orders in non-modifiable states
3. Stock is validated before confirming quantity changes
4. Order ownership verified before any mutation
5. Escalation triggered by all defined escalation conditions
6. Langfuse trace created for every invocation
7. SSE streaming works end-to-end
8. No prices generated by LLM — all from DB
